# FedProtoKD-IDS

Federated learning benchmark for network intrusion detection. Compares **FedProtoKD** (prototype-based knowledge distillation) against FedAvg, FedProx, and FedMD under non-IID conditions across three public IDS datasets.

**Thesis claim:** FedProtoKD converges faster and to higher accuracy than other aggregation methods under non-IID conditions, while maintaining low bandwidth usage.

## Setup

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

PyTorch `2.0.1` with CUDA is strongly recommended. CPU-only runs are supported but will be slow.

### 2. Download raw datasets

Place the raw CSV files in the corresponding `data/<dataset>/raw/` directories:

| Dataset | Source | Expected location |
|---|---|---|
| CIC-IDS-2018 | [UNB](https://www.unb.ca/cic/datasets/ids-2018.html) | `data/cic2018/raw/*.csv` |
| CIC-IDS-2017 | [UNB](https://www.unb.ca/cic/datasets/ids-2017.html) | `data/cic2017/raw/*.csv` |
| UNSW-NB15 | [UNSW](https://research.unsw.edu.au/projects/unsw-nb15-dataset) | `data/unswnb15/raw/*.csv` |

All scripts must be run from the **repository root**.

## Running the Experiment

Steps must be executed in order. Each step's output feeds the next.

### Step 1 — Preprocess CIC-IDS-2018

```bash
python src/datasets/preprocess_cic.py
```

Outputs to `data/cic2018/{processed_clean,processed_final,stats}/`.

### Step 2 — Preprocess CIC-IDS-2017

```bash
python src/datasets/preprocess_cic2017.py
```

Outputs to `data/cic2017/{processed_clean,processed_final,stats}/`.

### Step 3 — Preprocess UNSW-NB15

```bash
python src/datasets/preprocess_unswnb15.py
```

Outputs to `data/unswnb15/{processed_clean,processed_final,stats}/`.

Each preprocessing pipeline runs three stages internally:
1. **Stage 1** — stream raw CSVs, clean, map labels, write `.csv.gz` parts
2. **Stage 2** — compute global z-score stats and label encoders
3. **Stage 3** — apply normalisation, write final `.npz` files

### Step 4 — Partition data

```bash
python src/datasets/partition_data.py
```

Creates `partitions/<dataset>/train|test|public/` JSON manifests for three seeds (`42, 123, 456`) and partition types: Dirichlet α ∈ {0.1, 0.5, 1.0}, IID, and day-based.

### Step 5 — Run FL experiments

```bash
python scripts/train_baselines.py
```

Runs FedAvg, FedProx, FedProtoKD (and FedMD when a public set is available) across all datasets, models, partition types, and seeds. Saves results to `results/`.

## Environment Variables

You can scope a run to a single dataset or model to save time:

```bash
# Run only on CIC-IDS-2017 with the LSTM model
RUN_DATASET=cic2017 RUN_MODEL=lstm python scripts/train_baselines.py

# Use a different data root or results directory
DATA_ROOT=/data/mydata RESULTS_ROOT=/results/run1 python scripts/train_baselines.py
```

| Variable | Default | Description |
|---|---|---|
| `RUN_DATASET` | *(all)* | Restrict to one dataset (`cic2018`, `cic2017`, `unswnb15`) |
| `RUN_MODEL` | *(all)* | Restrict to one model (`lstm`, `mlp`, `cnn1d`, `transformer`) |
| `DATA_ROOT` | `.` | Root directory for `data/` and `partitions/` |
| `RESULTS_ROOT` | `results` | Output directory for figures and tables |

## Hyperparameters

Configured at the top of `scripts/train_baselines.py`:

| Parameter | Value | Notes |
|---|---|---|
| `num_rounds` | 20 | FL communication rounds |
| `clients_per_round` | 4 | Clients sampled per round |
| `batch_size` | 8192 | Local training batch size |
| `lr` | 3e-4 | Adam learning rate |
| `local_epochs` | 3 | Local training epochs per round |
| `lr_scheduler` | cosine | LR decay within each local training phase |

---

## Outputs

After `train_baselines.py` completes, `results/` contains:

- `figures/cm_<dataset>_<partition>_<model>_<strategy>.png` — confusion matrices (last seed)
- `figures/curves_<dataset>_<partition>_<model>.png` — accuracy / F1 / bandwidth curves
- `tables/table_<dataset>_<partition>_<model>.txt` — mean ± std summary table across seeds

Summary columns: Accuracy, F1-Macro, F1-Weighted, Rounds-to-90%, Total Bytes (MB).

---

## Strategies

| Strategy | Description |
|---|---|
| **FedAvg** | Weighted average of client model weights |
| **FedProx** | FedAvg + proximal regularisation (μ = 0.01) |
| **FedMD** | Knowledge distillation via shared public logits |
| **FedProtoKD** | CE loss + MSE to global per-class prototype (α = 0.3); bandwidth ≈ FedAvg |

FedMD requires a public (unlabelled) partition and is skipped automatically if none is found.

---

## Seeds

Three seeds are used (`42, 123, 456`). Final metrics are reported as **mean ± std** across seeds.
